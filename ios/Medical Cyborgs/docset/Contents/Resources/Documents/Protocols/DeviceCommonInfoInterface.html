DeviceCommonInfoInterface Protocol Reference

Conforms to CBPeripheralDelegate
Declared in DeviceCommonInfoInterface.h

Overview

This protocol is created to make an API for the devices so that similar operations can be performed. All of the devices are CBPeripheral devices. They have different UUIDs for their services and differing characteristics, so their interactions are different. This API is an attempt to hide the complexity for the sake of brevity in the code and to make things more readable elsewhere. Device specific operations are done in the class files associated with the devices.

batteryLevel

This method retrieves the battery level as expressed as a percent integer. It assumed that a updateBatteryLevel call was made, otherwise the last known value is returned.

- (int)batteryLevel

Return Value

NSInteger value from 0 to 100 expressing the percent of charge. 0 is default for unread or unreadable battery level characteristic.

initWithPeripheral:

This initialization method creates the desired device based on the CBPeripheral that is discovered. It is common to all types of devices and will bypass use of the standard init method for those classes. This method is used by the MonitorCreationFactory as a standard for initializing the device and providing all of the methods in this protocol.

- (id<DeviceCommonInfoInterface>)initWithPeripheral:(CBPeripheral *)peripheral

Parameters

peripheral

is the device object that the CBCentralManager discovered.

Return Value

id This object is any class that conforms to the DeviceCommonInfoInterface protocol.
