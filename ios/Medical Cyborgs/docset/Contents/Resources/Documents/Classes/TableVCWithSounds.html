Instance Methods

initWithDeviceManager:

Creates a device selection view controller. The purpose of the method is to create a standard initialization process for the subclasses of this class. All of the subclasses require a bluetooth device manager and need to be active to start building the view. The BTDeviceManager functions as the datasource for this tableviewcontroller and all the subclasses.

- (id)initWithDeviceManager:(BTDeviceManager *)newDeviceManager

Parameters

newDeviceManager

the bluetooth device manager that will serve as the data source.

Return Value

Returns a view controller unless no device manager is given, then the result is nil.

updateTable:

This function is the default method that is called when the device manager discovers a device. Its intent is to allow the tableview to update since the count of devices has changed and information is available. A table reload is used because the number of items to be discovered is very small and the overhead is small. Otherwise it is recommended to just reload the section and choose the type of animation.

- (void)updateTable:(NSNotification *)notification

Parameters

notification

A NSNotification object, not used at this time.
